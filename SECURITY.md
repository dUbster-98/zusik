# 보안 정책 (Security Policy)

zusik 은 **실거래(실제 자금) 자동매매 봇**입니다. 취약점은 금전 손실·계좌 탈취로 이어질 수
있으므로 보안을 최우선으로 다룹니다. 이 문서는 취약점 신고 방법과 현재 적용된 방어 체계를
설명합니다.

---

## 지원 버전 (Supported Versions)

| 버전 | 지원 |
|------|------|
| `master` (최신) | 유일하게 유지·패치되는 라인 |
| 그 외 과거 커밋 | 미지원 |

운영 서버는 Python 3.8.10 을 사용하며, CI 가 3.8~3.13 을 검증합니다.

---

## 취약점 신고 (Reporting a Vulnerability)

**공개 이슈(Issues)나 PR 로 취약점을 보고하지 마세요.** 공개되면 악용될 수 있습니다.

대신 다음 비공개 채널을 사용하세요:

1. **GitHub 비공개 취약점 신고** — 저장소 **Security 탭 → "Report a vulnerability"**
   (공개 저장소에서 활성화되는 기능. 비공개 저장소에서는 아래 2번 사용)
2. **메인테이너에게 직접 연락** — 저장소 소유자([@zusik-py](https://github.com/zusik-py))에게
   비공개로 연락 (이메일/DM). 보안 보고임을 제목에 명시해 주세요.

신고 시 포함해 주시면 좋은 것:
- 영향 범위(예: 주문 위·변조, 시크릿 노출, 원격 실행)와 재현 절차
- 영향받는 파일/커밋, 가능하면 PoC

**대응 약속**: 접수 후 가능한 빠르게(목표 72시간 내) 확인 회신하고, 수정 후 신고자에게
통지합니다. 책임 있는 공개(responsible disclosure)를 부탁드립니다 — 수정 배포 전 공개 자제.

## 적용된 방어 체계 (Security Measures)

이미 코드/CI 에 내장된 방어입니다:

- **주문 관문** (`OrderSafetyValidator`, `zusik/core/resilience.py`) — 모든 주문이 통과하는
  단일 관문. 전략·사이징 계층이 악의적으로 변형돼도 초과/유령 매도, 워시트레이딩,
  스푸핑 지정가, 현금 초과 매수, 과대수량을 **fail-closed** 로 차단.
- **코드 무결성 트립와이어** (`security_lock.py`) — `security_manifest.json` 기준선과 디스크
  코드를 sha256 비교. 봇 시작 시 자동 검증, `SECURITY_STRICT=true` 면 위반 시 기동 중단.
  CI 의 *무결성* 잡이 매 push 마다 재검증.
- **공급망 해시 핀** (`requirements.lock`) — 의존성을 sha256 으로 고정.
  `pip install --require-hashes -r requirements.lock` 로 변조 패키지 설치 차단.
- **CI 보안 검사** (`.github/workflows/`) — 매 push/PR 마다:
  - `gitleaks` 시크릿 스캔(블로킹) · `pip-audit` 의존성 CVE 감사 · 무결성 검증 · `ruff`
- **Dependabot** (`.github/dependabot.yml`) — 취약점 경보 + 보안 자동 PR + 주간 의존성 업데이트.
- **fail-closed 쿼터/한도 가드** — API 한도·검증 예외 시 안전한 방향(차단)으로 동작.

## 시크릿 취급 (운영자 책임)

- **시크릿(KIS 앱키/시크릿, Discord 토큰, Upbit 키)을 절대 커밋하지 마세요.**
  `.env` 는 `.gitignore` 로 제외되며, `main.py` 가 직접 로드합니다.
- 시크릿이 한 번이라도 노출되면(커밋/로그/화면 공유) **즉시 회전(재발급)** 하세요.
  git 히스토리에 남았다면 키 폐기가 유일한 안전책입니다.
- 모의투자(`KIS_VIRTUAL=true`)로 충분히 검증한 뒤 실거래로 전환하세요.
- 코드를 정식으로 변경한 뒤에는 `python3 security_lock.py generate` 로 무결성 기준선을
  갱신해 커밋하세요(그래야 트립와이어 오탐이 없습니다).

## 범위 (Scope)

- **대상**: 주문 위·변조, 인증/계좌 탈취, 시크릿 노출, 원격 코드 실행, 공급망 변조, 무결성 우회.
- **대상 아님**: 사용자 본인의 잘못된 설정(키 유출 등), 시장 리스크/투자 손실(자동매매 본질),
  제3자 서비스(KIS·Discord·Upbit) 자체의 취약점.
